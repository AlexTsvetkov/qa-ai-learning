# Module 6: Project — Production Monitoring Playbook

## 🎯 Objective
Create a complete production monitoring and debugging playbook for the QA team.

## Deliverables

### 1. Debugging Playbook
Step-by-step guides for debugging the 5 most common issues:
- Wrong answer (hallucination)
- Slow response
- Empty response / error
- Access control failure
- Missing/outdated information

Each guide: symptoms → Langfuse check → Grafana check → logs check → resolution.

### 2. Monitoring Dashboard Specification
Define the ideal QA monitoring dashboard: which panels, metrics, thresholds, alert rules.

### 3. Pre/Post-Release Checklist
Complete checklists for before and after each deployment.

### 4. Incident Response Template
Template for documenting and escalating AI platform incidents.

### 5. Bug Reports (3+ issues found during monitoring exercises)

## Evaluation Criteria
| Criteria | Weight |
|---------|--------|
| Debugging playbook quality | 30% |
| Dashboard specification | 25% |
| Release checklists | 25% |
| Incident template | 20% |

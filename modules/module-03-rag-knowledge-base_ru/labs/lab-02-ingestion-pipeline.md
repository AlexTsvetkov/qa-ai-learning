# Лаб 2: Тестирование ingestion-пайплайна

## 🎯 Цель
Протестировать полный цикл загрузки документов — от upload через обработку до поиска в Weaviate.

## ⏱️ Время: 2 часа

## Упражнение 1: Базовый тест ingestion (30 мин)
1. Создайте тестовый документ с уникальной фразой «QA_TEST_12345»
2. Загрузите через Ingestion API
3. Дождитесь обработки (проверьте статус задачи)
4. Найдите «QA_TEST_12345» в KB
5. Запишите время каждого этапа

## Упражнение 2: Граничные случаи ingestion (45 мин)

| # | Тест | Файл | Результат загрузки | Доступен в поиске? | Заметки |
|---|------|------|-------------------|-------------------|---------|
| 1 | Текстовый PDF | test.pdf | | | |
| 2 | Сканированный PDF | scan.pdf | | | Проверить OCR |
| 3 | Большой документ | 500стр.pdf | | | Таймаут? |
| 4 | Пустой файл | empty.pdf | | | Graceful error? |
| 5 | Битый файл | corrupt.pdf | | | Graceful error? |
| 6 | Повторная загрузка | test.pdf (снова) | | | Замена, не дубликат? |
| 7 | Unicode | russian.pdf | | | Кириллица? |

## Упражнение 3: Проверка чанков (30 мин)
Запросите Weaviate и проверьте: количество чанков, средний размер, разбиение по абзацам, целостность таблиц, перекрытие.

## Упражнение 4: Обновление документа (15 мин)
1. Загрузите V1 «Отпуск: 15 дней» → поиск → проверить «15»
2. Загрузите V2 «Отпуск: 20 дней» → поиск → должно быть «20»
3. V1 ещё виден? Должен быть заменён.

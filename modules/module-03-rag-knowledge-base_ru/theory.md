# Модуль 3: Теория — RAG и база знаний

## 3.1 Как работает RAG
1. **Embed** вопрос → вектор (HuggingFace TEI)
2. **Search** в Weaviate → найти похожие чанки + RBAC/ABAC фильтр
3. **Rank** результаты → Top-K
4. **Augment** промпт LLM → "На основе ЭТИХ документов ответь на вопрос"
5. **Generate** ответ → LLM выдаёт заземлённый ответ

## 3.2 Ingestion Pipeline
```
Документ → Ingestion API → Redis/RQ очередь → Worker:
  ├── Извлечение текста
  ├── OCR (PaddleOCR для сканов)
  ├── Чанкинг (разбиение на части)
  ├── Эмбеддинг (HuggingFace TEI)
  └── Сохранение в Weaviate
```

### Тест-кейсы ingestion
| Сценарий | Что проверяем |
|---------|--------------|
| Простой PDF | Все страницы проиндексированы |
| Большой PDF (500 стр.) | Не падает |
| Сканированный PDF | Качество OCR |
| Повторная загрузка | Замена, не дубликаты |
| Битый файл | Graceful error |

## 3.3 Качество retrieval
- **Golden Set** — набор пар вопрос-документ с известными правильными ответами
- **Метрики:** Precision, Recall, Relevance Score, Ranking Quality
- **Семантическое сходство:** разные формулировки → те же документы

## 3.4 Чанкинг
Плохой чанкинг = плохой retrieval. Проверяйте: размер чанков, разбиение по абзацам, таблицы целиком.

## 3.5 Каталог RAG-багов

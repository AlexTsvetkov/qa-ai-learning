# Лаб 2: Создание 14-дневного плана онбординга

## 🎯 Цель
Разработать практический 14-дневный план онбординга для нового Senior QA.

## ⏱️ Время: 3 часа

## Упражнение 1: Кастомизация плана (60 мин)

Для каждого дня укажите: конкретные URL, тестовые аккаунты, документы для проверки, вопросы для тестирования, ожидаемые первые баги.

## Упражнение 2: Чеклисты онбординга (45 мин)

Создайте чеклисты для дней 1, 3, 5, 7, 10, 14. Пример для Дня 1:
```
- [ ] Получен доступ к HR Assistant UI
- [ ] Получен доступ к Langfuse
- [ ] Получен доступ к Grafana
- [ ] Прочитана теория Модуля 1
- [ ] Могу объяснить поток данных от пользователя до ответа
- [ ] Отправил первый вопрос и нашёл трейс в Langfuse
```

## Упражнение 3: Гид по первым багам (30 мин)

Задокументируйте 10+ наиболее вероятных первых багов:

| # | Описание бага | Где найти | Как проверить | Модуль курса |
|---|-------------|----------|--------------|-------------|
| 1-10 | | | | |

## Упражнение 4: Критерии входа/выхода (45 мин)

| Фаза | Дни | Критерий входа | Критерий выхода | Метод оценки |
|------|-----|---------------|-----------------|-------------|
| Ориентация | 1-2 | Базовые QA навыки | Нарисовать архитектуру | Квиз + схема |
| Основное | 3-6 | Завершена ориентация | Тестировать агент, RAG, галлюцинации | Результаты лабов |
| Продвинутое | 7-10 | Основное пройдено | Безопасность, отладка через Langfuse | Баг-репорты |
| Самостоятельная работа | 11-14 | Продвинутое пройдено | Работать автономно | Финальный проект |

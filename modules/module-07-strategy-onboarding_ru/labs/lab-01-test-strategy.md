# Лаб 1: Создание документа тест-стратегии

## 🎯 Цель
Создать комплексный документ тест-стратегии для AI-платформы.

## ⏱️ Время: 3 часа

## Упражнение 1: Определение уровней тестирования (45 мин)

Для каждого уровня определите область, инструменты и типы тестов:

| Уровень | Область | Инструменты | Типы тестов | Кол-во кейсов |
|---------|---------|------------|------------|--------------|
| UI | React чат-интерфейс | Браузер, DevTools | Функциональные, UX | |
| API | .NET Backend | Postman, curl | Контрактные, валидация | |
| Агент | Agent Runtime, MCP | Langfuse | Поведенческие, границы | |
| AI Качество | Ответы LLM | Langfuse, Golden Set | Галлюцинации, GRACE | |
| Данные | KB, Weaviate, ingestion | KB API, Weaviate | Retrieval, ingestion | |
| Безопасность | RBAC/ABAC, инъекции | Мульти-роли | Контроль доступа | |
| Инфраструктура | vLLM, GPU, Redis | Grafana, Prometheus | Производительность | |

## Упражнение 2: Матрица рисков (30 мин)

Создайте матрицу с минимум 12 рисками: вероятность (1-5), влияние (1-5), скор, стратегия митигации.

## Упражнение 3: Критические сценарии (45 мин)

Определите минимум 8 обязательных сценариев (must-pass) с конкретными шагами, ожидаемыми результатами, инструментами проверки, частотой запуска.

## Упражнение 4: Сборка документа стратегии (60 мин)

Объедините всё в единый документ: Обзор, Уровни тестирования, Матрица рисков, Критические сценарии, AI-специфичный подход, Инструменты, Метрики, Процесс релиза.

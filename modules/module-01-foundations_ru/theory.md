# Модуль 1: Теория — Основы AI для QA

## 1.1 Что такое AI, ML и LLM?

### Искусственный интеллект (AI)

**Искусственный интеллект** — область компьютерной науки, направленная на создание систем, способных выполнять задачи, требующие человеческого интеллекта.

Для QA-инженера важно понимать: AI — это не магия, а набор алгоритмов и моделей, которые:
- Обучаются на данных
- Находят паттерны
- Генерируют ответы на основе этих паттернов

### Машинное обучение (ML)

**Machine Learning** — подраздел AI, где системы обучаются на данных, а не программируются явно.

**Аналогия для тестировщика:**
- Традиционное программирование: вы пишете правила → система их выполняет
- ML: вы даёте примеры → система сама выводит правила

```python
# Традиционный подход — правила написаны вручную
def is_bug_critical(bug):
    if bug.crash_count > 10 and bug.affected_users > 100:
        return True
    return False

# ML подход — модель обучена на исторических данных
# model.predict(bug_features) → critical / not critical
```

### Большие языковые модели (LLM)

**Large Language Models** — нейронные сети, обученные на огромных объёмах текста. Они умеют:
- Понимать и генерировать текст
- Следовать инструкциям
- Писать и анализировать код
- Рассуждать о задачах

**Ключевые LLM для QA:**

| Модель | Компания | Особенности |
|--------|----------|-------------|
| GPT-4o | OpenAI | Универсальная, отлично пишет код |
| Claude | Anthropic | Отличный анализ длинных документов |
| Gemini | Google | Интеграция с Google-экосистемой |
| Copilot | GitHub/Microsoft | Специализируется на коде |

> ⚠️ **Важно для QA**: LLM не «понимают» код в человеческом смысле. Они генерируют статистически вероятные ответы. Ответы нужно **всегда проверять**.

## 1.2 Обзор AI-инструментов для QA

### Универсальные AI-ассистенты

- **ChatGPT** — генерация тест-кейсов, написание автотестов, анализ кода
- **Claude** — анализ больших документов, детальный code review
- **GitHub Copilot** — автодополнение кода тестов в реальном времени

### Специализированные AI-инструменты для тестирования

| Инструмент | Назначение | Тип |
|------------|-----------|-----|
| **Testim** | AI-powered UI тестирование | Self-healing тесты |
| **Applitools** | Визуальное AI-тестирование | Сравнение скриншотов |
| **Mabl** | Интеллектуальная автоматизация | Low-code тестирование |
| **Katalon** | AI-assisted создание тестов | Комплексная платформа |

## 1.3 Как AI помогает в тестировании

| Задача | Без AI | С AI |
|--------|--------|------|
| Написание тест-кейсов | 2-4 часа | 30 мин + ревью |
| Code review тестов | 1-2 часа | 20 мин |
| Анализ падения тестов | 30 мин - 2 часа | 5-15 мин |
| Создание тестовых данных | 1-3 часа | 15 мин |
| Документация | 2-4 часа | 30 мин + редактура |

## 1.4 Ограничения и риски использования AI в QA

### 🚫 Основные ограничения

1. **Галлюцинации** — AI может генерировать правдоподобный, но неверный контент
2. **Отсутствие контекста проекта** — AI не знает специфику вашего приложения
3. **Устаревшие знания** — модели обучены на данных до определённой даты
4. **Конфиденциальность** — не отправляйте секретные данные в публичные AI-сервисы

> ⚠️ **ВАЖНО**: Не отправляйте в публичные AI-сервисы: проприетарный код, персональные данные, API-ключи, пароли, конфиденциальную бизнес-информацию.

### Правильный подход: Human-in-the-Loop

**AI — это инструмент усиления, а не замена QA-инженера.**

## 1.5 Ключевые термины

| Термин | Определение |
|--------|-------------|
| **AI** | Системы, имитирующие человеческий интеллект |
| **ML** | Обучение на данных без явного программирования |
| **LLM** | Большая языковая модель (GPT, Claude) |
| **Prompt** | Запрос/инструкция для AI |
| **Token** | Единица текста (~3/4 слова) |
| **Context Window** | Максимальный объём текста для обработки за раз |
| **Hallucination** | Генерация правдоподобного, но неверного контента |
| **Temperature** | Параметр «креативности» (0 = точный, 1 = креативный) |

## 📖 Итоги модуля

1. **AI** — это инструмент, а не замена тестировщика
2. **LLM** (ChatGPT, Claude) — наиболее полезные AI для QA прямо сейчас
3. AI может значительно **ускорить** рутинные задачи QA
4. Всегда **проверяйте** выходные данные AI
5. Соблюдайте **безопасность** и конфиденциальность данных

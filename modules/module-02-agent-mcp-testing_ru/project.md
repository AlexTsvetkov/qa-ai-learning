# Модуль 2: Проект — Набор тестов агента и MCP

## 🎯 Цель
Создать полноценный набор тестов для поведения агента и MCP-оркестрации.

## ⏱️ Время: 3 часа

## Результаты

### 1. Тест-сьют выбора инструментов (15+ тест-кейсов)

| Категория | Мин. кейсов | Фокус |
|----------|------------|-------|
| Правильный выбор инструмента | 5 | Вопрос → нужный инструмент |
| Обнаружение неверного выбора | 3 | Вопросы, провоцирующие ошибку |
| Сценарии без инструментов | 3 | Приветствия, вне области |
| Мульти-инструмент | 2 | Вопросы, требующие несколько инструментов |
| Граничные случаи | 2 | Неоднозначные, пустые, длинные |

Каждый тест-кейс должен включать: ID, описание, входной вопрос, ожидаемый инструмент, как проверить (Langfuse трейс).

### 2. Тест-план обработки отказов MCP (10+ тест-кейсов)
- MCP-сервер недоступен
- Таймаут
- Битый JSON в ответе
- Частичный отказ в цепочке
- Восстановление после сбоя

### 3. Тест-сьют промпт-инъекций (8+ тест-кейсов)
- Прямое переопределение инструкций
- Смена роли
- Попытка извлечь системный промпт
- Попытка извлечь данные
- Многоходовая манипуляция

### 4. Баг-репорты (минимум 3 бага)

## Критерии оценки

| Критерий | Вес |
|---------|-----|
| Полнота покрытия | 30% |
| Реалистичность тест-кейсов | 25% |
| Качество анализа Langfuse | 20% |
| Качество баг-репортов | 15% |
| Документация | 10% |

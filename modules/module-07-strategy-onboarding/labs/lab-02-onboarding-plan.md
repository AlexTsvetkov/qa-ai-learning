# Lab 2: Creating the 14-Day Onboarding Plan

## 🎯 Objective
Design a practical 14-day onboarding plan for a new Senior QA joining the project.

## ⏱️ Estimated Time: 3 hours

---

## Exercise 1: Customize the Onboarding Template (60 min)

Using the template from theory, customize it for your actual environment:

For each day, specify:
- Exact URLs to access (platform, Langfuse, Grafana)
- Specific test accounts to use
- Specific documents/policies to verify against
- Specific questions to test with
- Expected first bugs and where to find them

---

## Exercise 2: Create Onboarding Checklists (45 min)

### Day 1 Checklist (New QA fills this out)
```
- [ ] Got access to: HR Assistant UI
- [ ] Got access to: Langfuse dashboard
- [ ] Got access to: Grafana dashboard
- [ ] Got access to: API documentation
- [ ] Read: Module 1 theory
- [ ] Can explain the data flow from user to response
- [ ] Sent first question and found the trace in Langfuse
```

Create similar checklists for Day 3, Day 5, Day 7, Day 10, Day 14.

## Exercise 4: Create Entry/Exit Criteria (45 min)

Define what "done" looks like for each onboarding phase:

| Phase | Duration | Entry Criteria | Exit Criteria | Assessment Method |
|-------|----------|---------------|---------------|-------------------|
| Orientation | Days 1-2 | QA basics, system access | Can draw architecture, found first trace | Quiz + diagram |
| Core Testing | Days 3-6 | Completed orientation | Can test agent, RAG, find hallucinations | Lab deliverables |
| Advanced | Days 7-10 | Core testing done | Can test security, debug with Langfuse | Bug reports |
| Independent | Days 11-14 | Advanced done | Can work independently, created strategy | Final project |

# Module 4: Quiz — AI-Powered Analysis & Debugging

---

### Q1: What is a key advantage of using AI for log analysis?
a) AI can fix bugs automatically  
b) AI can identify patterns across thousands of lines quickly  
c) AI always finds the correct root cause  
d) AI can access production systems directly  

### Q2: What is the "5 Whys" technique?
a) Asking 5 different questions about the same bug  
b) Iteratively asking "why" to trace a problem to its root cause  
c) Having 5 people review the same bug  
d) Running a test 5 times to check for flakiness  

### Q3: When asking AI to analyze logs, what is most important to provide?
a) The complete application source code  
b) Context about the system architecture and the reported issue  
c) The user's personal information  
d) The database schema  

### Q4: What makes a test "flaky"?
a) It always fails  
b) It always passes  
c) It passes and fails intermittently without code changes  
d) It takes too long to run  

### Q5: Which is NOT a good use of AI in bug analysis?
a) Categorizing bug reports  
b) Suggesting root causes  
c) Making final decisions on whether to fix a bug  
d) Correlating events across multiple log files  

---

<details>
<summary>Нажмите для ответов</summary>

1. **b)** AI can identify patterns across thousands of lines quickly
2. **b)** Iteratively asking "why" to trace a problem to its root cause
3. **b)** Context about the system architecture and the reported issue
4. **c)** It passes and fails intermittently without code changes
5. **c)** Making final decisions on whether to fix a bug — AI assists, humans decide

</details>

# Module 4: Project — Bug Investigation Report

## 🎯 Objective
Perform a comprehensive AI-assisted bug investigation and produce a professional report.

## ⏱️ Время: 3 часов

---

## Scenario

You are given a production incident:

**Incident**: E-commerce platform experienced a 40% increase in failed orders during a 2-hour window (10:00-12:00 UTC). No deployments were made during this period. The system has: React frontend, Java Spring Boot backend, PostgreSQL database, Redis cache, and Stripe payment integration.

**Available Data** (create realistic simulated data):
1. Application server logs (create sample logs showing the issue)
2. Database slow query logs
3. 5 related bug reports from users
4. Test results from automated regression suite (showing new failures)

## Результаты

### 1. Simulated Data (`exercises/module04_incident_data/`)
Create realistic log files and bug reports for the scenario.

### 2. AI-Assisted Analysis (`exercises/module04_analysis.md`)
- Log analysis summary
- Timeline of events
- 5 Whys root cause analysis
- Bug classification and prioritization

### 3. Investigation Report (`exercises/module04_report.md`)
Professional incident report with: summary, timeline, root cause, impact, fix recommendations, and prevention measures.

## Критерии оценки

| Criteria | Points |
|----------|--------|
| Realistic simulated data | 20 |
| Thorough AI-assisted analysis | 30 |
| Professional investigation report | 30 |
| Prevention recommendations | 20 |
| **Total** | **100** |

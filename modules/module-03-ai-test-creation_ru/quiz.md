# Module 3: Quiz — AI-Assisted Test Creation

---

### Q1: What should you always do after AI generates test cases?
a) Use them as-is  
b) Review, validate, and refine them  
c) Delete them and start over  
d) Only run the positive test cases  

### Q2: Which type of test data is AI particularly good at generating?
a) Production database dumps  
b) Edge cases and boundary values  
c) Encrypted data  
d) Performance benchmarks  

### Q3: What is a pairwise testing approach?
a) Testing with two users simultaneously  
b) Comparing two versions of the application  
c) Covering all combinations of pairs of input variables  
d) Running each test twice  

### Q4: When converting manual tests to automated tests with AI, what should you NOT do?
a) Specify the target framework  
b) Trust selectors/locators without verification  
c) Provide context about the application  
d) Review the generated code  

### Q5: What format should you request for machine-processable test data?
a) Plain text paragraph  
b) JSON or CSV  
c) Screenshot  
d) PDF  

### Q6: What is adversarial test data?
a) Data from competitors  
b) Data designed to break or exploit the application  
c) Random data  
d) Production data copy  

### Q7: Why should you avoid `time.sleep()` in AI-generated tests?
a) It's not valid Python  
b) It makes tests slow and flaky  
c) It costs more API calls  
d) It's a security risk  

### Q8: What is a test coverage gap analysis?
a) Finding untested requirements and scenarios  
b) Measuring code coverage percentage  
c) Counting the number of test cases  
d) Analyzing test execution time

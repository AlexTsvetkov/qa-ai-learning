# Модуль 4: Теория — Тестирование LLM

## 4.1 Типы галлюцинаций

| Тип | Описание | Пример | Критичность |
|-----|---------|--------|-------------|
| **Фабрикация** | Выдумывает несуществующие факты | "Согласно политике HR-099..." (нет такой) | Critical |
| **Противоречие** | Противоречит документу-источнику | Документ: 15 дней, LLM: 20 дней | Critical |
| **Экстраполяция** | Добавляет детали не из источника | Документ: "отпуск доступен", LLM добавляет "до 6 месяцев" | Major |
| **Ошибка атрибуции** | Приписывает не тому источнику | Цитирует финансовый документ для HR-политики | Major |
| **Конфляция** | Смешивает информацию из разных документов | Объединяет отпуск и больничный | Minor |

### Как обнаруживать галлюцинации
1. Каждый факт в ответе прослеживается до документа-источника?
2. Числа совпадают точно?
3. Ответ добавляет информацию, которой НЕТ в документах?
4. Ответ противоречит документам?

## 4.2 Фреймворк GRACE для оценки качества

| Измерение | Описание |
|-----------|---------|
| **G**roundedness (Заземлённость) | Основан ли на документах? |
| **R**elevance (Релевантность) | Отвечает ли на заданный вопрос? |
| **A**ccuracy (Точность) | Факты верны? |
| **C**ompleteness (Полнота) | Покрывает все аспекты? |
| **E**xpression (Выражение) | Хорошо структурирован и понятен? |

## 4.3 LLM-специфичные отказы
- **Переполнение контекстного окна** — слишком много документов → обрезка → деградация качества
- **Недетерминизм** — факты должны быть одинаковыми, формулировки могут различаться
- **Проблемы формата** — markdown в plain-text, обрезанный ответ, не тот язык

## 4.4 Тестирование системных промптов
Промпты управляются в Langfuse (реестр с версионированием). Тестируйте: границы области, инъекции, цитирование документов, обработку языков.

## 4.5 Тестирование эндпоинтов LLM Gateway
- `/chat` — генерация ответов (temperature, max_tokens, конкурентность)
- `/embed` — эмбеддинги (детерминизм, схожесть, edge cases)
- `/ocr` — OCR (качество, повороты, многоязычность, числа)

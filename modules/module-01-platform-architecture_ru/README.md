# Модуль 1: Архитектура платформы и основы AI для QA

## 📋 Обзор модуля

Этот модуль знакомит с корпоративной AI-платформой, которую вам предстоит тестировать. Вы изучите полную архитектуру системы — от запроса пользователя до ответа AI — и поймёте ключевые AI-концепции (LLM, RAG, эмбеддинги, агенты) с точки зрения QA. Предыдущий опыт с AI не требуется.

## 🎯 Цели обучения

После завершения модуля вы сможете:
- Нарисовать полный поток данных: User → FE → BE → Agent → MCP → KB → LLM → Response
- Объяснить назначение каждого компонента (Agent Runtime, MCP, LLM Gateway, Knowledge Base, Ingestion Pipeline)
- Определить, где системы обычно ломаются и что QA должен тестировать на каждом уровне
- Понимать ключевые AI-концепции (LLM, RAG, эмбеддинги, галлюцинации) без глубокой ML-теории
- Описать 3 релиза платформы и их область (HR, Finance, Clerk)

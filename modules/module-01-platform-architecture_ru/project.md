# Модуль 1: Проект — Шпаргалка по архитектуре для QA

## 🎯 Цель
Создать справочный документ (1-2 страницы), который любой QA-инженер сможет использовать для быстрого понимания системы.

## Требования
1. **Схема системы** — все компоненты и связи
2. **Таблица компонентов** — технология, назначение, эндпоинты, что тестируем
3. **«Где сломалось?»** — симптом → куда смотреть → как проверить
4. **Ключевые URL** — все ссылки для ежедневной работы QA
5. **Глоссарий** — 15 важнейших терминов

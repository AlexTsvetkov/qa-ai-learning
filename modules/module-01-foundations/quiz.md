# Module 1: Quiz — AI Foundations for QA

Test your understanding of the key concepts from this module.

---

## Questions

### Q1: What is the correct hierarchy of AI concepts?
a) ML ⊂ AI ⊂ DL ⊂ LLM  
b) AI ⊂ ML ⊂ DL ⊂ LLM  
c) LLM ⊂ DL ⊂ ML ⊂ AI  
d) DL ⊂ LLM ⊂ AI ⊂ ML  

### Q2: What is a "hallucination" in the context of AI?
a) When AI refuses to answer a question  
b) When AI generates plausible but incorrect content  
c) When AI takes too long to respond  
d) When AI copies text verbatim from the internet  

### Q3: Which of the following should you NEVER send to a public AI service?
a) Generic test case templates  
b) API keys and passwords  
c) Questions about testing methodologies  
d) Public documentation  

### Q4: What does "temperature" control in an LLM?
a) The speed of response generation  
b) The length of the response  
c) The creativity/randomness of the response  
d) The cost of the API call  

### Q5: What is the "Human-in-the-Loop" approach?
a) AI replaces humans completely  
b) Humans review and verify AI outputs before using them  
c) Only humans do the work, no AI involved  
d) AI monitors human performance  

### Q6: Which AI tool is best known for its large context window (200K+ tokens)?
a) GitHub Copilot  
b) ChatGPT  
c) Claude  
d) Gemini  

### Q7: What is a "token" in the context of LLMs?
a) An authentication credential  
b) A unit of text (approximately 3/4 of a word)  
c) A payment unit for API usage  
d) A type of encryption key  

### Q8: Which statement about AI in QA is TRUE?
a) AI can fully replace QA engineers today  
b) AI-generated tests never need review  
c) AI is a force multiplier for QA engineers  
d) AI understands your project context automatically  

### Q9: What is a key limitation of LLMs for QA work?
a) They can only write code in Python  
b) They have no knowledge of testing  
c) Their training data has a cutoff date  
d) They cannot process text inputs  

### Q10: What is the recommended first step when using AI-generated test code?
a) Deploy it to production immediately  
b) Run it without reviewing  
c) Review and understand what it does before using  
d) Delete it and write from scratch  

---

## Answers

<details>
<summary>Click to reveal answers</summary>

1. **c)** LLM ⊂ DL ⊂ ML ⊂ AI — LLMs are a subset of Deep Learning, which is a subset of Machine Learning, which is a subset of AI.

2. **b)** When AI generates plausible but incorrect content — hallucinations look convincing but contain errors.

3. **b)** API keys and passwords — never share secrets with public AI services.

4. **c)** The creativity/randomness of the response — 0 = deterministic, 1 = creative.

5. **b)** Humans review and verify AI outputs before using them — AI assists, humans validate.

6. **c)** Claude — Anthropic's Claude is known for its large context window.

7. **b)** A unit of text (approximately 3/4 of a word) — tokens are how LLMs process text.

8. **c)** AI is a force multiplier for QA engineers — it enhances, not replaces.

9. **c)** Their training data has a cutoff date — models may not know about recent changes.

10. **c)** Review and understand what it does before using — always verify AI outputs.

</details>
